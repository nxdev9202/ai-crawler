# 02. 24절기 절입시각 데이터 (1900–2100)

> 데이터 위치: `../data/solar-terms/solar_terms_1900_2100.json` (+ 동일 내용 CSV)
> 생성 스크립트: `../data/solar-terms/generate_solar_terms.py`

## 왜 필요한가

사주의 **월주(月柱)는 음력 달이 아니라 절기력 기준**으로 정해진다. 예컨대 입춘~경칩 사이가 인(寅)월.
따라서 "몇 월 며칠 몇 시에 절기가 바뀌는가"(절입시각)를 **분 단위**로 알아야 경계일 출생자의 월주를 정확히 세울 수 있다.
또한 **연주(年柱)도 입춘 시각 기준**으로 바뀐다 (음력 설날 기준 아님).

## 데이터 생성 방법

- KASI 공개 API는 절기 "날짜"만 제공하고 시·분은 제공하지 않으므로, 천문 계산으로 자체 생성했다.
- 방법: pyephem으로 태양의 **시황경(apparent ecliptic longitude, epoch of date)** 이 15° 배수를 통과하는 순간을 초 단위 이진 탐색.
  - 절기의 공식 정의(태양 황경 15° 간격) 그대로이며, KASI가 쓰는 정의와 동일.
  - 구현 주의점: `ephem.Ecliptic(sun)`은 astrometric 좌표(a_ra) 기준이라 **약 10분 오차**가 난다.
    반드시 apparent 좌표(`g_ra/g_dec`)를 epoch-of-date 황도좌표로 변환해야 함 (스크립트에 반영됨).

## 정확도 검증

| 절기 | 본 데이터 | 비교값 | 차이 |
|---|---|---|---|
| 2024 입춘 | 2024-02-04 17:26:56 KST | KASI 공표 17:27 | **약 4초** |
| 1955 입추 | 1955-08-08 16:50:13 (UTC+9 환산) | rgbitcode 블로그 인용 "16:14" | 아래 참고 |

**1955년 값 차이에 대한 정직한 기록**: 블로그가 인용한 "16:14"는 시중 만세력 앱들이 쓰는 값인데,
당시 한국 표준시는 UTC+8:30이었다 (1954-03-21~1961-08-09). 본 데이터의 UTC 절입시각(07:50 UTC)을
UTC+8:30으로 환산하면 16:20이 되어 여전히 약 6분 차이가 있다. 이는 구형 만세력들이 과거 역서(曆書)의
저정밀 절기표를 쓰기 때문으로 추정된다. 2024년 검증에서 KASI 공표값과 4초 이내로 일치했으므로
**천문학적으로는 본 데이터가 더 정확**하다. 다만 "시중 만세력 앱과 결과를 똑같이 맞추는 것"이 목표라면
경계 사례에서 수 분 차이가 날 수 있음을 인지할 것 (연간 절기 경계 ±10분 이내 출생자에게만 영향).

## 스키마

```json
{
  "kst": "2024-02-04 17:26:56",   // 절입시각 (UTC+9 고정)
  "utc": "2024-02-04 08:26:56",   // 절입시각 (UTC) — 과거 표준시 변환은 이 값 기준으로
  "year": 2024,
  "sun_longitude": 315,            // 태양 황경 (도)
  "name": "입춘",
  "is_jeol": true,                 // true = 절(節): 월주가 바뀌는 12개 절기
  "month_branch": "인(寅)"          // 이 절기부터 시작되는 사주 월지 (절만 해당)
}
```

- 총 4,824건 (201년 × 24절기)
- `is_jeol=true`인 12개(입춘·경칩·청명·입하·망종·소서·입추·백로·한로·입동·대설·소한)만 월주 경계.
  기(氣) 12개(우수·춘분 등)는 참고용.

## 사용 규칙

1. **월주 결정**: 출생시각(보정 완료된 표준시)이 어느 절입시각 구간에 속하는지로 월지 결정.
2. **연주 결정**: 출생시각 < 그해 입춘 시각 → 전년도 간지.
3. **과거 표준시 주의**: 1954-03-21~1961-08-09 출생자는 시계가 UTC+8:30이므로,
   비교 전에 출생시각을 UTC로 환원한 뒤 `utc` 필드와 비교하는 것이 안전 (`03-서머타임-기간표.md`의 timezone history 참조).
4. 재생성이 필요하면: `python3 -m venv venv && venv/bin/pip install ephem && venv/bin/python generate_solar_terms.py`
